# Hybrid-PURIFY-OD: strongest generic detox pipeline

This is the next-stage detox path for the case where ordinary ASR-aware hard-sample fine-tuning reduces internal ASR but fails on external hard suites, or where strong ASR reduction destroys clean mAP.

## Method

Hybrid-PURIFY-OD combines:

1. **External hard-suite closed-loop selection**: external ASR dominates checkpoint selection because internal regression can be self-consistent.
2. **External replay**: high-ASR external attack datasets are replayed with correct labels.
3. **Phase-separated optimization**: OGA, ODA, semantic, WaNet and clean recovery are trained separately.
4. **PGBD-style prototype sanitization**: target-present regions align with clean target prototypes; target-absent images suppress target-prototype similarity.
5. **I-BAU-style adversarial unlearning**: small unknown perturbations are generated by PGD on detection loss, then optimized against.
6. **NAD / feature / output distillation**: a clean teacher preserves normal detector behavior and avoids mAP collapse.

## Recommended command

```powershell
python scripts/hybrid_purify_detox_yolo.py `
  --model "D:\clean_yolo\best 2.pt" `
  --teacher-model "D:\clean_yolo\trusted_clean_teacher.pt" `
  --images "D:\clean_yolo\dataset\images\train" `
  --labels "D:\clean_yolo\dataset\labels\train" `
  --data-yaml "D:\clean_yolo\dataset\data.yaml" `
  --target-classes helmet `
  --external-eval-roots "D:\clean_yolo\poison_benchmark_cuda_tuned" `
  --external-replay-roots "D:\clean_yolo\poison_benchmark_cuda_large" `
  --out "D:\clean_yolo\model_security_gate\runs\hybrid_purify_best2" `
  --cycles 4 `
  --phase-epochs 2 `
  --feature-epochs 2 `
  --recovery-epochs 2 `
  --max-allowed-external-asr 0.10 `
  --max-map-drop 0.03 `
  --device 0
```

For final reporting, keep external replay roots and external eval roots separate so the selected checkpoint is measured on held-out hard suites.

## Success criteria

A candidate is not acceptable unless:

- external max ASR <= 10%;
- internal max ASR <= 10%;
- clean mAP50-95 drop <= 3 points;
- badnet_oda is not worse than baseline;
- semantic and WaNet ASR improve;
- acceptance_gate passes with `--attack-metrics` and `--safety-critical`.

## Notes

If `teacher_model` is omitted, the pipeline falls back to a frozen copy of the suspicious model for distillation. That is weaker and should be treated as risk reduction only. For best results, train or provide a clean teacher from trusted weights and audited clean data.
